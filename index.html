Settings
Access Protection (end-to-end encryption)
When enabled, all portal data is encrypted with a passphrase before being stored locally or synced to the Gist. Uses AES-256-GCM with PBKDF2 key derivation (120,000 iterations). The passphrase is held only in memory and never transmitted.
🔒 Protection ON
Team Deployment
Generate a customized index.html file with your Gist ID and an encrypted version of the token baked in. Team members who open this file only need the passphrase — they never have to configure Gist ID or token. Ideal for client-facing deployments.
Shared Storage (optional)
Configure a GitHub Gist to sync data across team members. All users with this Gist ID and Token will read/write the same data. Token is stored in your browser only.
This portal is running in Team Deployment mode. Gist ID and token are baked into the HTML and decrypted by the passphrase. End users never need to touch these fields.
